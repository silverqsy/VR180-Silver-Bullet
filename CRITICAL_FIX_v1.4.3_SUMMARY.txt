🐛 THE PROBLEM
================================================================================

Windows users reported this error when trying to import videos:

  Command '['Y:\...\ffprobe.EXE', '-v', 'quiet', '-select_streams',
  'v:0', '-show_entries', 'stream=width,height', '-show_entries',
  'format=duration', '-of', 'json', 'Y:\...\video.mov']'
  returned non-zero exit status 1.

Impact:
  ❌ Video import completely broken
  ❌ App unusable on Windows
  ❌ FFprobe couldn't find bundled executable
  ❌ All video processing failed

Root Cause:
  - Path detection looked for 'ffprobe' without .exe extension
  - Windows requires 'ffprobe.exe' (with extension)
  - Path('ffprobe').exists() returned False
  - Even though ffprobe.exe was present in bundle

================================================================================
✅ THE FIX
================================================================================

Added platform-specific executable name detection:

BEFORE (Broken):
  ffprobe_path = base_path / 'ffprobe'  # ❌ Fails on Windows

AFTER (Fixed):
  ffprobe_name = 'ffprobe.exe' if sys.platform == 'win32' else 'ffprobe'
  ffprobe_path = base_path / ffprobe_name  # ✅ Works on Windows

Applied to 3 functions:
  ✅ get_ffmpeg_path() - Line 67
  ✅ get_ffprobe_path() - Line 102
  ✅ get_spatial_path() - Line 137

================================================================================
🔧 TECHNICAL DETAILS
================================================================================

Why shutil.which() Wasn't Affected:
  - shutil.which() automatically adds .exe on Windows
  - Path.exists() does NOT add .exe automatically
  - This is why system FFmpeg worked but bundled didn't

Platform Detection:
  - sys.platform == 'win32' detects Windows 10/11
  - sys.platform == 'darwin' detects macOS
  - Correct for all Python versions

PyInstaller Bundle Structure:
  Windows: _internal/ffprobe.exe (with extension)
  macOS: Resources/ffprobe (without extension)

💡 LESSONS LEARNED
================================================================================

1. Platform-Specific Testing Critical
   - Must test on Windows AND macOS
   - Can't assume cross-platform compatibility
   - Executable extensions differ by platform

2. Path Detection Edge Cases
   - Path.exists() doesn't auto-add extensions
   - shutil.which() does (causing confusion)
   - Always test bundled vs system executables

3. Emergency Patch Process
   - Quick identification: Error message showed ffprobe.EXE
   - Fast fix: Simple extension handling
   - Rapid release: Same-day critical fix

4. Prevention
   - Add Windows-specific integration tests
   - Test PyInstaller bundles on both platforms
   - Check all executable path detections
