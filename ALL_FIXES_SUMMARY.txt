FIX #1: FFmpeg Binary Path Resolution
──────────────────────────────────────
Files Modified: vr180_gui.py, vr180_cli.py

Added Functions:
  • get_ffmpeg_path() - Finds ffmpeg in bundle or system
  • get_ffprobe_path() - Finds ffprobe in bundle or system

Search Order:
  1. PyInstaller _MEIPASS directory (runtime extraction)
  2. macOS .app bundle Resources folder
  3. macOS .app bundle Frameworks folder
  4. System PATH (fallback)

Changed:
  • All "ffmpeg" strings → get_ffmpeg_path()
  • All "ffprobe" strings → get_ffprobe_path()

Result:
  ✅ Bundled binaries are now found correctly
  ✅ Works on systems without FFmpeg installed
  ✅ Falls back to system FFmpeg during development

FIX #2: Progress Monitoring Non-Blocking I/O
────────────────────────────────────────────
File Modified: vr180_gui.py

Problem:
  • readline() blocked forever waiting for data
  • FFmpeg had finished but UI didn't know

Solution:
  1. Check process.poll() FIRST (is FFmpeg done?)
  2. Use select() with 0.5s timeout (macOS/Linux)
  3. Fall back to readline() on Windows
  4. Call process.wait(timeout=5) after loop
  5. Always emit 100% when returncode == 0

Result:
  ✅ No more hanging at random percentages
  ✅ Progress reaches 100% correctly
  ✅ Clean process termination
  ✅ "Complete!" dialog appears properly
