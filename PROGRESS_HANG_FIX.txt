PROGRESS HANG FIX - Processing Completion Detection
================================================================================

PROBLEM:
────────
When processing a 3:49 ProRes clip, the app:
  - Got stuck at 35% with nothing moving
  - Had to be force closed
  - BUT the file was actually finished processing

This is a classic "blocking I/O" issue.

ROOT CAUSE:
───────────
The original code used `readline()` to read progress from FFmpeg:

  while True:
      line = process.stdout.readline()  # <-- BLOCKS FOREVER
      if not line and process.poll() is not None: break
      ...

The problem:
  1. FFmpeg finishes encoding and closes
  2. BUT there might be buffered data in the pipe
  3. readline() blocks waiting for a newline that never comes
  4. The UI hangs even though the file is complete

SOLUTION IMPLEMENTED:
────────────────────
1. Check process.poll() FIRST to see if FFmpeg has exited
2. Use select() with timeout on Unix/Mac to avoid blocking
3. Add process.wait(timeout=5) to ensure clean termination
4. Track last_progress to avoid duplicate updates

KEY CHANGES:
────────────
✓ Check if process.poll() is not None BEFORE reading
✓ Use select() with 0.5s timeout on macOS/Linux
✓ Windows: Continue using readline (select not available)
✓ Call process.wait(timeout=5) after loop exits
✓ Emit 100% progress when process.returncode == 0

CODE FLOW:
──────────
Before:
  1. Read line (blocks if no data)
  2. Check if process finished
  3. Repeat

After:
  1. Check if process finished (FIRST!)
  2. If not, try to read with timeout
  3. Update progress if we got data
  4. Repeat

This ensures the UI never blocks waiting for data that won't come.

BENEFITS:
─────────
✓ No more hanging at 35% (or any percentage)
✓ Properly detects when FFmpeg finishes
✓ Progress bar reaches 100% correctly
✓ Clean process termination
✓ Works on both macOS and Windows
